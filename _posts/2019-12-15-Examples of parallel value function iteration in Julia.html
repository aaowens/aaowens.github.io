Serial¶

Here we perform a serial (single-core) value function iteration to solve a simple savings problem with idiosyncratic income shocks. To solve the Bellman equation at each point in the state space, we use numerical optimization (Brent's method) and monotonic cubic spline interpolation over the expected value function. For simplicity we iterate a fixed 100 times.

# Bellman step given evf and x
function bellman(evf, x)
    vf, pol = similar(evf), similar(evf)
    for i in eachindex(vf)
        a = agrid[i]
        obj = ap -> -(u(a, ap, x) + beta * evf(ap))
        upper = min(maxap(a, x), agrid[end])
        sol = optimize(obj, agrid[1], .99*upper)
        vf[i], pol[i] = -sol.minimum, sol.minimizer
    end
    return vf, pol
end

function expectation(bigvf, P)
   evf = similar(bigvf)
    for a_nx in 1:size(bigvf, 1)
        for x_nx in 1:size(P, 1)
        ev = 0.
            for xp_nx in 1:size(P, 2)
                ev += bigvf[a_nx, xp_nx] * P[x_nx, xp_nx]
            end
            evf[a_nx, x_nx] = ev
        end
    end
    return evf
end

Threads¶

Here we use multithreading. See https://docs.julialang.org/en/v1/manual/parallel-computing/#Setup-1 for how to enable this. There is no interaction between loop iterations, so this is thread-safe. We check the output at the end.

pmap¶

Now we use distributed memory parallelism with pmap. This requires that the code is loaded on all nodes. The easiest way to do this is to define a module everywhere.

vfall3 = zeros(length(DM.agrid), length(DM.xgrid))
polall = zeros(length(DM.agrid), length(DM.xgrid))
function test_pmap(vfall, polall)
    fill!(vfall, 0)
    fill!(polall, 0)
    for i in 1:100
        vfe = DM.expectation(vfall, DM.P)
        vfolds = [DM.interpolate(DM.agrid, vfe[:, x_nx], DM.SteffenMonotonicInterpolation()) for x_nx in eachindex(DM.xgrid)]
        vfs_pols = pmap(DM.bellman, vfolds, DM.xgrid)
        for x_nx in eachindex(DM.xgrid)
            vfall[:, x_nx] = vfs_pols[x_nx][1]; polall[:, x_nx] = vfs_pols[x_nx][2]
        end
    end
    return vfall, polall
end

Threads is about 4x faster than serial with 4 threads, while pmap is only about 2x faster. The relative slowness of pmap improves with the size of the task. Each call to the bellman function only takes about 0.5 ms here, so the overhead from the distributed memory nature of pmap is significant. For larger state spaces the bellman function will use more time, and its performance approaches that of threads in my testing.

Julia can make use of multiple nodes on a cluster, and pmap will distribute the work across available nodes automatically if you make use of the ClusterManagers package. For example, if you have a Slurm cluster with 2 nodes and 4 tasks per node, the following code should set up and confirm the communication is working.
